Implementations

Wetherell Shannon (WS)

The algorithm specified by Wetherell and Shannon has been implemented, and the source code can be found in the directory "source/WetherellShannon"
The Implementation has support for both variations of the algorithm mentioned in the paper i.e

  • The NORMAL implementation which centers parents over it's children but is not minimum in width
  • The MODIFIED implementation which provides minimum width but may produce 'UGLY' trees due to the absence of centering
    The trees that have been generated (both on user input and otherwise) and used for WS binary search trees.
    See also
    mainWS.c WetherellShannon.h

Tilford Reingold (TR)

The algorithm specified by Tilford and Reingold has been implemented and the source code can be found in the directory "source/TilfordReingold". again the tree type that has been used for this implementation of TR is the binary search tree.

See also

Extension to N-ary Trees

Using the previous two papers as inspiration we have extended the main concept of the WS Algorithm to accomodate arbtrarily size trees.
We use a modifed version of the tree node used in WS by adding sibling pointers. The algorithm performs two walks similar to WS. Before performing an addional pre order walk to draw the tree with the generated coordinates.

Drawing Algorithms

We have used 3 fundamental algorithms to accomplish the rendering part of the project.

  1. Generalised Bresenham's Line Drawing
  2. Midpoint Circle Algorithm Using 8-Way symmetry
  3. Variation of 4-Neighbour Boundary Fill
    Starting from circle's center a 4 neighbour fill is applied while the point has not been visited and does not lie outside the circle'
