Wetherell Shannon

1. Description and Time Efficiency

AESTHICS

1. Nodes of a tree at the same height should lie along a straight line, and the straight lines defining the level should be parallel.

2. In a binary tree, each left son should be positioned left of its father and each right son right of its father.

3. A parent should be centered over its children.


ALGORITHM

Computing Y coordinate

  • Each node's y-coordinate is a multiple of its height.
  • This is saved as the yPos attribute in our code

Computing X coordinate

  • Array : next_pos stores the next available node position for each level in the tree.
  • If a leaf at level h is under consideration, placing the leaf at next_pos [h] is legal and satisfies the minimum width requirement.
  • Integer place stores ,for an internal node, the average of its sons' positions
  • The actual position assigned is the maximum of the place and next_pos [h], since sons may try to drag their father too far to the left and cause the father to collide with his relative to the left.
  • For each node store the distance to the place in a modifier field. During the first pass, which assigns positions a modifier is kept for each level.
  • This is saved as the xPos attribute in our code

Producing Minimum Width Tree (may violate Aesthetic 3)

  • In the second while loop ,pass variable modifier_sum in a pre order walk .
  • maintain for each tree level the actual next position available at the leaf
  • Final position of a node is the minimum of the next available position at the node's level, its left son's position plus one, its father's position plus one (for a right son), and the position that would have been applied by above algorithm .
  • The MODIFIED second pass implements this variation of the algorithm


Mathematical Analysis of Time Complexity

Let : n(T)= number of nodes in the tree.

h(T)= height of the tree.

The time required by the algorithm depends on the two while loops.

F(T)= the number of times the body of the loop is executed for a tree T.

F(T)=F(L1)+F(L2)

Here,

L1= while loop for determining X coordinate

L2= while loop for modifying X coordinates to produce minimum width tree

F(L1)= It traverses through the enter tree. (n(T))

F(L2)= traverses through the enter tree. (n(T))

Hence , F(T)= 2n(T)

3. Memory / Space Analysis

Let h(T)= height of the tree =h n(T)=numbers of nodes in the tree= n

According to the algorithm,

  • Size of Array next_pos[ ]= h(T)
  • Size of Array modifier[]= h(T)
  • For other variables = Constant space size required= k

Total space required= 2*h(T)+k = O(h(T)+k) =O(h) Hence the algorithm has O(n)- Linear Space Complexity for additional storage

In addition to this, space is also required to store the nodes of the tree. Each node of the tree l according to the paper ,contained the following:

data: integer;
father branch;
left_son, right_son branch;
height : integer;
x, y integer;
status: (first_visit, left_visit, right_visit); modifier : integer;

Hence total space occupied by a node (Assuming 4 byte pointers and integer data) is 4*4 +4*3.

Hence for a tree with n nodes, the space complexity is = O(28*n(T))= O(n) Linear

Hence total space complexity for Wetherell Shannon is 2*O(n) = O(n)
